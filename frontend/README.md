# 🔊 Sound Guard System (음향 기반 위험 예방·구조 시스템) - Dashboard Demo

본 프로젝트는 상황실 관리자가 특정 구역의 음향 데이터를 실시간으로 모니터링하고, 무단침입 및 응급 상황 발생 시 즉각적으로 대응할 수 있도록 돕는 **관제 대시보드 프론트엔드 프로토타입**입니다.

실제 백엔드 서버 및 AI 음향 인식 모델이 연동되기 전, 클라이언트 및 이해관계자에게 **시스템의 핵심 워크플로우와 UX/UI를 직관적으로 시각화하여 전달**하기 위해 단일 React 컴포넌트로 구현된 시뮬레이션 데모입니다.

---

## 🚀 시작하기 (How to Run)

본 데모는 React 환경에서 바로 실행해 볼 수 있습니다. 프로젝트 폴더로 이동한 후 아래 명령어를 순서대로 입력하세요.

1. 의존성 패키지 설치
\`\`\`bash
npm install
\`\`\`

2. 로컬 개발 서버 실행 (데모 구동)
\`\`\`bash
npm run dev
\`\`\`

## 📑 주요 화면 및 세부 기능 설계 의도

### 1. 🔐 로그인 화면 (Login Screen)
시스템 접근 권한을 확인하는 첫 관문으로, 관제 시스템 특성상 무거운 느낌을 주면서도 신뢰감을 주도록 다크 테마 기반으로 설계되었습니다.

* **기능 상세**
    * **ID/PW 인증:** `admin` / `1234` 하드코딩을 통한 데모 계정 인증 지원.
    * **비밀번호 가시성 토글:** '표시/숨김' 버튼을 제공하여 사용자 편의성 증대.
    * **비동기 로딩 시뮬레이션:** 로그인 버튼 클릭 시 0.6초(`600ms`)의 지연(Timeout)을 주어 실제 서버와 통신하는 듯한 피드백(UX) 제공.
    * **에러 핸들링:** 잘못된 정보 입력 시 즉각적인 붉은색 경고창 표시.
* **데모 구현 의도:** 실제 서비스와 동일한 진입 경험을 제공하여 프로토타입의 완성도를 높이고, 보안이 중요한 관제 시스템이라는 인식을 심어주기 위함입니다.

### 2. 🖥️ 메인 대시보드 화면 (Main Screen)
로그인 직후 바로 진입하게 되는 시스템의 핵심 화면입니다. 한눈에 모든 상황을 파악할 수 있도록 좌우 2단 그리드 레이아웃(280px : 1fr)으로 촘촘하게 설계되었습니다.

* **글로벌 컨트롤 (Header & Footer):** 상단에서 현재 관리 구역명, 서버 접속 IP, 관리자 ID를 확인합니다. 여기서 **'안내 멘트 설정'** 버튼을 통해 초기 설정 화면으로 진입할 수 있습니다.
* **좌측 패널 (상태 요약 & 로그):**
    * **현재 상태 배너:** 정상(초록) / 무단침입(노랑) / 위험감지(빨강) 3가지 상태에 따라 배경색과 아이콘이 전환됩니다.
    * **구역 정보:** 구역명에 마우스 오버 시 전체 박스 배경색이 하이라이트 되며 상호작용(구역 변경)이 가능함을 직관적으로 알립니다.
    * **이벤트 로그:** 시스템의 모든 변화를 시간 역순으로 로깅합니다.
* **중앙 패널 (실시간 분석 및 시각화):**
    * **지도 시각화 (Map):** 레이더망 형태의 그리드 배경 위에 사고 발생 지점을 핑(Ping) 애니메이션으로 시각화합니다.
    * **실시간 음향 분석 (BEATs):** 발소리, 사람 음성, 비명/충격음, 환경음 4가지 요소의 확률값을 실시간 프로그레스 바 형태로 시각화합니다.
* **데모 구현 의도 (랜덤 시뮬레이션):** 실제 AI 모델이 없어도 `setInterval`을 이용해 7초마다 특정 확률(정상 43%, 1차 29%, 2차 14%, 응급 14%)로 가상의 음향 데이터를 생성합니다. 이를 통해 프로토타입 시연 시 가만히 두어도 대시보드가 살아서 움직이는 듯한 역동성을 부여했습니다.

### 3. 📝 안내 멘트 설정 화면 (Config Screen)
메인 대시보드 상단의 '안내 멘트 설정 ➡️ 초기 설정' 메뉴를 클릭하여 진입하는 화면입니다. 현장 상황에 맞게 시스템 전체의 방송 멘트를 능동적으로 세팅합니다.

* **기능 상세**
    * **구역명 설정:** 관리할 구역의 이름을 지정.
    * **단계별 경고 멘트 커스텀:** 1차 경고(5초 체류), 2차 경고(15초 체류), 응급 상황에 대한 송출 메시지를 각각 다르게 입력.
    * **실시간 미리보기 (Preview):** 멘트 작성 시 `"이곳은 [구역명] 입니다."`라는 공통 접두사가 자동으로 결합된 최종 송출 문구를 실시간으로 하단에 렌더링.
* **데모 구현 의도:** 사용자가 시스템에 끌려가는 것이 아니라 현장에 맞게 **능동적으로 시스템을 제어할 수 있다는 점을 어필**하며, 미리보기 기능을 통해 실제 방송될 문구의 어감을 직관적으로 파악해 입력 실수를 방지하도록 UX를 설계했습니다.

### 4. 🎛️ 모달 팝업 화면 (Modals)
화면 전환 없이 메인 대시보드 위에서 즉각적인 수정이 가능하도록 오버레이 팝업을 활용했습니다.

* **기능 상세:**
    * **안내 멘트 분기 메뉴:** 설정 버튼 클릭 시 멘트를 완전히 '초기 설정'할지, 모니터링을 유지하며 '멘트 수정'만 할지 선택하는 팝업.
    * **수정 오버레이:** 1/2차 경고문 및 응급 멘트를 즉석에서 수정 및 적용.
    * **구역 관리 모달:** 구역 정보 패널 클릭 시 프리셋 구역을 선택하거나 새 구역을 추가/삭제.
    * **지도 위치 설정:** 지도의 좌표를 클릭하여 GPS 좌표 및 주소를 수동 맵핑.
* **데모 구현 의도:** 관제 중 화면이 완전히 전환되면 시야를 잃게 되므로, **모니터링의 연속성을 해치지 않기 위해 (Context Switch 방지)** 간단한 설정 변경은 모두 모달(Modal) 형태로 처리했습니다.

## 🛠 기술 스택 및 구조 특이점 (Technical Notes)

* **Framework:** React 18+ (Functional Components & Hooks)
* **Styling:** Inline CSS & Styled Objects (`const C = {...}`)
    * *이유:* 별도의 CSS 파일이나 Tailwind 같은 라이브러리 설치 없이, **단일 파일(`.jsx`)만 복사하여 어디서든 즉시 실행하고 배포**할 수 있도록 이식성을 극대화하기 위해 인라인 스타일을 채택했습니다.
* **State Management:** `useState`, `useRef`
    * *이유:* `setInterval` 시뮬레이션 내부에서 최신 상태값을 누락 없이 참조하기 위해 상태(Status) 변수들을 `useRef`로 동기화하여 클로저(Closure) 문제를 해결했습니다.
* **No Dependencies:** 외부 아이콘 라이브러리나 무거운 차트 라이브러리 없이 유니코드(⚠, ✓, 📢 등)와 순수 CSS(keyframes)만을 사용하여 가볍고 빠른 렌더링을 보장합니다.
